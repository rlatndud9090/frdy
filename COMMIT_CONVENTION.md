# Git Commit Convention

이 프로젝트는 **Conventional Commits** 기반의 커밋 컨벤션을 사용합니다.

## 기본 형식

```
<type>(<scope>): <subject>

[본문 - 선택사항]

[꼬리말 - 선택사항]
```

## Type (필수)

게임 개발에 특화된 타입:

| Type | 설명 | 예시 |
|------|------|------|
| `feat` | 새 기능 추가 | `feat(combat): 턴제 전투 시스템 구현` |
| `fix` | 버그 수정 | `fix(map): 노드 연결 오류 수정` |
| `art` | 아트/그래픽 작업 | `art: 용사 스프라이트 추가` |
| `audio` | 사운드/음악 작업 | `audio: 전투 배경음악 추가` |
| `balance` | 게임 밸런스 조정 | `balance(card): 마력 소모량 조정` |
| `ui` | UI/UX 변경 | `ui: 의심 수치 게이지 디자인` |
| `refactor` | 코드 리팩토링 | `refactor(map): 노드 생성 로직 개선` |
| `test` | 테스트 추가/수정 | `test(combat): 전투 승리 조건 테스트` |
| `docs` | 문서 작업 | `docs: 게임 컨셉 업데이트` |
| `chore` | 기타 작업 | `chore: .gitignore 업데이트` |

## Scope (선택사항)

게임 시스템별 분류:

| Scope | 설명 |
|-------|------|
| `combat` | 전투 시스템 |
| `map` | 맵/노드 시스템 |
| `card` | 카드 시스템 |
| `hero` | 용사 관련 |
| `enemy` | 적 관련 |
| `event` | 이벤트 시스템 |
| `suspicion` | 의심 시스템 |
| `ui` | UI 요소 |
| `core` | 핵심 엔진/프레임워크 |

## Subject (필수)

- 50자 이내로 작성
- 마침표 없음
- 명령형으로 작성 ("추가함" ❌ → "추가" ✅)
- 한글 사용

## 예시

### 좋은 커밋 메시지

```
feat(combat): 턴제 전투 시스템 구현

용사와 적이 번갈아가며 공격하는 기본 전투 로직 추가.
- 턴 관리 시스템
- 공격력/체력 계산
- 전투 종료 조건
```

```
fix(suspicion): 의심 수치 계산 오류 수정

조력 행동 시 의심 수치가 감소하던 버그 수정
```

```
balance(card): 마왕 카드 마력 소모량 조정

- 치유 카드: 3 → 2 마력
- 공격 방해 카드: 2 → 3 마력
```

```
art: 용사 기본 스프라이트 추가
```

### 나쁜 커밋 메시지

```
fix: 버그 수정  ❌ (무엇을 고쳤는지 불명확)
```

```
feat: 기능 추가했습니다  ❌ (명령형 아님, 구체적이지 않음)
```

```
update  ❌ (타입이 규칙에 없음, 설명 부족)
```

## AI 커밋 가이드

AI가 커밋을 생성할 때:
1. 변경사항을 분석하여 적절한 type 선택
2. 해당되는 scope가 있으면 추가
3. 50자 이내의 명확한 subject 작성
4. 복잡한 변경사항은 본문에 상세 설명 추가
